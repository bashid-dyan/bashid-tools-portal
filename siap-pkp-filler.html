SIAP PKP Filler

by Bashid Effendi

Tools untuk mengisi kolom Hasil Pengujian pada halaman Prosedur PKP di SIAP BPK secara otomatis dari file Excel. Hemat waktu, kurangi pekerjaan repetitif.

Download SIAP PKP Filler (.zip)

Versi terbaru — Membutuhkan Python 3.8+ dan koneksi VPN BPK

Prasyarat

  • ✓

    Python 3.8+ terinstall di komputer. Download di python.org. Saat install, centang "Add Python to PATH".

  • ✓

    Browser Chrome untuk mengakses SIAP BPK dan export data.

  • ✓

    VPN BPK terhubung. Tools ini hanya bisa berjalan saat komputer terhubung ke jaringan BPK.

Instalasi (Sekali Saja)

1

Extract file ZIP yang sudah didownload ke folder manapun (misal Desktop).

2

Buka folder hasil extract, lalu double-click Install.bat. Tunggu sampai selesai. Ini hanya perlu dilakukan sekali.

Cara Penggunaan

A. Siapkan File Excel PKP

1

Download file Excel PKP dari SIAP BPK: buka menu Perencanaan → Dokumen Pemeriksaan → Program Kerja Perseorangan → klik Detail → pilih dokumen atas nama Anda → Lihat File → Download.

2

Tambahkan kolom "Hasil Pemeriksaan" di kolom G pada file Excel tersebut. Anda dapat memanfaatkan Claude for Excel untuk membantu mengisi Hasil Pemeriksaan berdasarkan data CALK, SAP, atau dokumen pendukung lainnya. Cukup upload dokumen ke Claude for Excel dan minta untuk menghasilkan narasi Hasil Pemeriksaan per prosedur.

3

Jika tidak berlangganan Claude, Anda tetap dapat mengisi kolom Hasil Pemeriksaan secara manual sesuai hasil pengujian yang telah dilakukan.

4

Simpan file Excel.

B. Export Data dari Browser

1

Login ke SIAP BPK menggunakan browser Chrome.

2

Buka halaman Pelaksanaan → Prosedur pada sub pemeriksaan yang ingin diisi. Pastikan semua prosedur tampil (filter dalam keadaan kosong/default).

3

Di folder SIAP PKP Filler, cari file export_dari_browser.js. Klik kanan file tersebut → Open with → Notepad. Lalu tekan Ctrl+A (Select All) dan Ctrl+C (Copy).

Jangan double-click file JS-nya! Harus dibuka pakai Notepad lalu copy isinya.

4

Kembali ke halaman SIAP BPK di Chrome. Tekan F12 (atau Fn+F12 di laptop HP) untuk membuka panel Developer Tools. Klik tab Console.

5

Klik pada area Console, lalu tekan Ctrl+V untuk paste.

Jika muncul pesan "Warning: Don't paste code...", ketik allow pasting lalu tekan Enter. Setelah itu baru paste ulang (Ctrl+V) dan tekan Enter.

6

Tunggu hingga muncul pesan "SELESAI". Dua file akan otomatis terdownload: auth.json dan prosedur_list.json.

C. Jalankan Tools & Upload

1

Pastikan VPN BPK terhubung.

2

Double-click Jalankan.bat. Akan muncul jendela hitam (CMD) — jangan ditutup selama menggunakan tools.

3

Buka Chrome, ketik di address bar: localhost:5001 lalu Enter.

4

Upload ketiga file: auth.json, prosedur_list.json, dan file Excel PKP. Isi Nama Sheet, NIP, dan Nama Auditor.

5

Disarankan pilih Dry Run (Simulasi) terlebih dahulu untuk memastikan data yang akan diisi sudah benar. Jika sudah yakin, jalankan ulang dengan mode Submit Langsung.

6

Tunggu proses selesai. Setelah selesai, Anda dapat menutup jendela CMD.

Catatan Penting

Hasil belum divalidasi. Tools ini hanya mengisi kolom Hasil Pengujian tanpa melakukan validasi. Anda masih dapat mengubah isi, melengkapi narasi, atau mengunggah file pendukung langsung di SIAP. Validasi sengaja tidak dilakukan otomatis agar Anda memiliki kesempatan mengecek kembali sebelum finalisasi.

Auth token bersifat sementara. File auth.json hanya berlaku selama sesi login browser Anda aktif. Jika Anda logout atau menutup browser, export ulang auth.json dengan menjalankan script di Console lagi.

Bisa untuk pemeriksaan berbeda. Cukup buka halaman Prosedur di sub pemeriksaan yang berbeda, jalankan export ulang, lalu jalankan tools dengan file Excel yang sesuai.

Pertanyaan Umum

Error "Python is not recognized"?

Python belum terinstall atau belum masuk PATH. Install ulang Python dan pastikan mencentang "Add Python to PATH" saat instalasi.

Error "No module named flask/openpyxl"?

Jalankan Install.bat lagi untuk menginstall library yang dibutuhkan.

Auth expired / Error 401?

Login ulang ke SIAP BPK di Chrome, lalu jalankan script export_dari_browser.js di Console lagi untuk mendapatkan auth.json yang baru.

Apakah bisa dipakai auditor lain?

Ya! Setiap auditor cukup mengisi NIP dan Nama masing-masing di form. File auth.json dan prosedur_list.json harus di-export dari browser masing-masing auditor.

Kenapa harus dijalankan lokal? Tidak bisa online?

Karena website SIAP BPK hanya bisa diakses dari jaringan internal BPK (via VPN). Server online tidak bisa menjangkau SIAP, sehingga tools ini harus berjalan di komputer yang terhubung VPN BPK.
